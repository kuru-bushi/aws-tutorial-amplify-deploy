# 開発ルール詳細（Claude 作業ガイド）

このファイルは `CLAUDE.md`（インデックス）から参照される詳細ルール集です。
CLAUDE.md には要点のみを置き、具体的な運用ルールはこのファイルに記載する。

## コード編集ルール（学習目的）

このプロジェクトはユーザーのチュートリアル学習を兼ねるため、ユーザー自身が手を動かすことを重視する。

- **Claude は `my-amplify-app/` 配下のアプリのソース（`src/` / `amplify/` / 各種設定ファイル）を編集しない。**
  - `Edit` / `Write` で直接書き換えないこと。
- 変更が必要なときは、次の形式で**指示のみ**を行う：
  1. **編集するファイルのパス**（例: `my-amplify-app/src/App.tsx`）
  2. **編集箇所**（どこを・どう変えるか。新規追加 / 置き換え / 削除を明示）
  3. **ターミナルに貼り付け用のコードブロック**を提示（ユーザーがコピーして自分で編集）
- 実際の編集は**ユーザーが手動で行う**。

### 例外（Claude が編集してよいもの）

- ユーザーが「直接編集して」と**明示指示した場合**の `my-amplify-app/` 配下。
- ドキュメント類（`my-amplify-app/docs/`、プロジェクトルートの `docs/` / `index.md` / `task_board.md` / `memo` / `CLAUDE.md` など）。

### 読み取りについて

- 読み取り・調査・状態確認は通常どおり行ってよい（制約の対象は「編集」のみ）。

## ドキュメント配置ルール

- **amplify-app（アプリ本体）に関するドキュメントは `my-amplify-app/docs/` に置く。**
- プロジェクト全体の運用ルール・作業ガイドはプロジェクトルートの `docs/` に置く（このファイルがその一例）。
- ドキュメントを新規作成・移動したら、`CLAUDE.md` の「ドキュメント在りか一覧」を必ず更新する。
