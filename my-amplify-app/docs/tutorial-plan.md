# チュートリアル進め方・完了条件

このプロジェクト（`my-amplify-app`）は AWS Amplify Gen 2 の学習を兼ねる。
本ドキュメントはチュートリアルの**ゴール・完了条件・手順**を示す。

## ゴール（完了条件）

> **フロントとバックエンドを最小限だけ編集して連携させ、本番環境にデプロイできることを確認できたら完了とする。**

具体的には、次がすべて満たされた状態を「完了」とする。

1. バックエンド（Amplify Gen 2 の `auth` / `data`）がクラウドにデプロイされている。
2. フロント（React + Vite）がバックエンドに接続し、最小機能（Todo の表示・追加）が動く。
3. **本番環境（Amplify Hosting）にデプロイされ、本番 URL 上で同じ動作が確認できる。**

「最小限の編集」を重視し、機能の作り込みは目的としない（デプロイの一連の流れを通すことが目的）。

## 進め方（手順）

| # | ステップ | 内容 | 編集対象 |
|---|----------|------|----------|
| 1 | sandbox 起動 | `npx ampx sandbox` を起動しっぱなしにし、クラウドにバックエンドをデプロイ。`amplify_outputs.json` が生成される | （編集なし） |
| 2 | バックエンド最小編集 | `data/resource.ts` の `Todo` モデルを利用（既定のまま。必要なら `isDone` 追加程度） | `amplify/data/resource.ts` |
| 3 | フロント配線 | `Amplify.configure(outputs)` を追加し、フロントが接続情報を読むようにする | `src/main.tsx` |
| 4 | フロント最小実装 | `generateClient<Schema>()` で Todo の一覧表示・追加を実装 | `src/App.tsx` |
| 5 | ローカル確認 | `npm run dev` で Todo の表示・追加が動くか確認 | （確認のみ） |
| 6 | 本番デプロイ | GitHub にプッシュ → Amplify Hosting でリポジトリ接続 → ビルド & デプロイ | （Hosting 設定） |
| 7 | 本番確認 | 発行された本番 URL でフロント表示・Todo 動作を確認 → **完了** | （確認のみ） |

## 編集の進め方（重要）

- コードの編集は**ユーザー自身が手動で行う**。Claude は編集箇所とコピペ用コードを提示するのみ。
- 詳細は [`../../docs/development-rules.md`](../../docs/development-rules.md) を参照。

## 進捗の記録

- 各ステップの進捗・状態は `task_board.md`（プロジェクトルート）に記録する。
